The Problem

Pokemon Auto Chess is a competitive game where players need to make probability-based decisions about which units to pursue ‚Äî but the game provides no analytics. Players were left guessing at odds, manually tracking what opponents had taken, and making suboptimal decisions due to incomplete information.

What I Built

A Chrome extension that reverse-engineers the game client to extract live game state, processes a 408kb data file containing 1,060+ entities with evolution chains and rarity mappings, and calculates real-time probability distributions across all 8 players with pool depletion tracking. The tool runs geometric distribution math to show exact odds of finding target units, expected rolls to hit confidence thresholds, and warns when targets become contested or impossible.

Live Interface

Main overlay panel with live pool tracking, target selection, evolution family counting, and probability calculations updating in real-time

Team Tracker: Multi-target tracking with per-roll probability, expected rolls to hit confidence threshold, and combined probability across all targets

Live gameplay view with minimized overlay showing pool state ‚Äî expands on demand for full probability analysis

Configurable polling speeds from 250ms to 5s, balancing update frequency with performance
